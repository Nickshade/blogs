Git 命令

YAOHAN-LI
2025-04-13

Git 提交规范

    [Add]           新增内容，功能、文件、依赖等
    [Fix]           修复Bug或错误行为
    [Update]        更新已有功能
    [Remove]        删除代码、功能、文件等
    [Refactor]      代码重构
    [Docs]          文档改动，如README或注释等
    [Style]         格式修改(不影响功能), 如缩进
    [Build]         构建系统或依赖相关更新

diff 命令

作用

    比较文件差异(用于行比较)

参数

    -u:输出带有上下文

例子

    diff -u <file1> <file2>
    diff -u <file1> <file2> > diff.txt

patch命令

作用

    apply a diff file to an original

例子

    patch <file2> < diff.txt
    patch -R <file1> < diff.txt

git了解

    暂存区: stage/index(.git/index)
    .git/index: 实际上是一个包含文件索引的目录树,像是一个虚拟的工作区,记录着文件名和文件的状态信息
    .git/objects/ : Git对象库,保存文件的内容

git命令

查看git版本:

    git --version

init

    1.创建版本库(在当前目录)
            git init
    2.创建版本库(指定目录)
            git init <dir>

config

作用

    获取或设置git配置文件(git配置文件采用 INI 文件格式)

option

    --local :  编辑 .git/config 版本库配置文件(默认)
    --global:  编辑 /home/username/.gitconfig 全局配置文件
    --system:  编辑 /etc/gitconfig 系统级配置文件
    配置生效优先级 local > global > system

使用

    1、打开文件进行编辑
            git config -e [--local | -- global | --system]
    2、读取配置文件内容
            git config <section>.<key>
    3、更改或设置配置文件内容
            git config <section>.<key> <value>
    4、删除配置文件内容
            git config --unset <section>.<key>
    5、设置命令别名
            git config alias.<别名>  "<命令>"
            eg: git config alias.ci "commit -s"
    6、操作其它的 INI 格式文件
            GIT_CONFIG=file.ini git config <section>.<key> <value>

commit

作用

    将暂存区提交到版本库

参数

    --amend: 对刚刚的提交进行修补
    --allow-empty: 允许空白提交
    --reset-author: 修改提交者

使用

    1、允许执行空白提交
            git commit --allow-empty -m "commit message"
    2、修改上次未设置 user 和 email 时的提交
            git commit --allow-empty --reset-author

ls-tree

作用

参数

    -l : 显示文件大小,单位字节
    -r : 递归显示目录内容
    -t : 递归遇到的每棵树都显示出来,而不止显示最终文件

用例

    1、查看版本库的目录树
            git ls-tree -l HEAD
            eg: $100644 blob fdhefislefjlsfeoaw76a7eae 25 file.txt
            字段说明
                    第一个字段: 文件属性
                    第二个字段: 代表一个 blob 对象(文件)
                    第三个字段: 文件在对象库中的对应ID
                    第四个字段: 文件大小
                    第五个字段: 文件名
    2、查看暂存区的目录树
            (1)、先将暂存区的目录树写入git对象库中
                    git write-tree
                    eg: c31b7457bea68e2f16c602e9490d76d054a5122f
            (2)、针对目录树执行
                    git ls-tree -l c31b745

ls-files

作用

参数

用例

    1、显示暂存区的目录树
            git ls-files -s
            eg: $100644 fdhefislefjlsfeoaw76a7eae 0 file.txt
            字段说明
                    第一个字段: 文件属性
                    第二个字段: 文件在对象库中的对应ID
                    第三个字段: 暂存区编号
                    第四个字段: 文件名

rm

作用

参数

用例

    1、删除暂存区中的文件,工作区不受影响
            git rm --cached <file>

clean

作用

参数

用例

    1、清楚非跟踪的文件和目录
            git clean -fd

status

作用

    查看工作树状态

参数

用例

    1、显示精简的状态输出
            git status -s
            eg: [MM] welcome.txt
            第一列 M :版本库与暂存区的文件比较
            第二列 M :工作区与暂存区的文件比较

log

作用

参数

    --stat: 打印每次提交时文件变更统计

使用

    1、查看详细的提交日志
            git log --pretty=fuller
    2、精简输出日志
            git log --pretty=oneline

diff

作用

    参看文件差异(工作区/暂存区/版本库)

参数

使用

    1、比较 工作区 与 暂存区 差异
            git diff <path>
    2、比较 工作区 与 版本库(HEAD/master/other branch)
            git diff HEAD
            git diff master
    3、比较 暂存区 与 版本库
            git diff [--cached | --staged]

stash

作用

参数

用例

    1、保存 工作区 与 暂存区 的改动(不包括未跟踪的文件)
            git stash

rev-parse

作用

    通常用于解析 Git 引用并返回相应的提交哈希值

使用

    1、显示版本库 .git 目录所在位置
            git rev-parse --git-dir
    2、显示工作区根目录
            git rev-parse --show-toplevel
    3、相对于工作区根目录的相对目录
            git rev-parse --show-prefix
    4、显示从当前目录(cd)后退(up)到工作区的根的深度
            git rev-parse --show-cdup

生词

    1、stage: 阶段、状态
    2、discard: 丢弃
    3、blob: 一小片、一滴
