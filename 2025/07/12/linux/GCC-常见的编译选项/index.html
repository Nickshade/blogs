GCC 常见的编译选项

YAOHAN-LI
2025-07-12

GCC（GNU Compiler Collection）常用的编译选项非常丰富，下面按照用途分类，介绍最常见、最实用的编译参数。

✅ 一、基本编译流程相关

选项 含义
-c 只编译，不链接（生成 .o 文件）
-o <file> 指定输出文件名
-E 只做预处理，输出预处理结果（.i）
-S 只编译到汇编代码（生成 .s 文件）
-v 显示编译详细过程
--help 显示所有选项帮助

✅ 二、警告与错误

选项 含义
-Wall 开启大多数警告（推荐）
-Wextra 开启额外警告
-Werror 将所有警告视为错误
-Wno-xxx 禁用某个具体警告，如 -Wno-unused
-pedantic 遵循标准 C 语法规范，报出非标准用法

✅ 三、优化选项（用于发布或性能测试）

选项 含义
-O0 无优化（默认，适合调试）
-O1 一般优化
-O2 更高级别优化（推荐生产使用）
-O3 更激进优化（包括循环展开等）
-Os 优化代码尺寸（适合嵌入式）
-Ofast 放弃部分标准兼容性换取性能
-flto 启用链接时优化（Link Time Optimization）

✅ 四、调试选项

选项 含义
-g 生成调试信息（用于 GDB 调试）
-g3 包含宏定义等更详细调试信息
-ggdb 为 GDB 优化调试信息

✅ 五、链接选项

选项 含义
-l<lib> 链接库，如 -lm 表示链接 libm.so
-L<dir> 指定库搜索路径
-static 静态链接全部库
-shared 编译为共享库（动态库 .so）
-Wl,-rpath,<path> 嵌入动态库搜索路径到 ELF 的 rpath 中
-Wl,--start-group ... --end-group 解决多个库间循环依赖

✅ 六、输出控制 & 编译阶段

选项 含义
-M 生成头文件依赖（用于 Makefile）
-MF 指定依赖输出文件
-nostdlib 不链接系统标准库
-nostartfiles 不使用启动文件（嵌入式场景）

✅ 七、语言/标准指定

选项 含义
-std=c99 使用 C99 标准（可换为 c89、c11、gnu11 等）
-ansi 等价于 -std=c89 -pedantic
-x c / -x c++ 指定输入文件语言类型（用于无扩展名文件）

✅ 八、宏定义与包含路径

选项 含义
-D<macro> 定义宏，如 -DDEBUG 或 -DVER=1
-U<macro> 取消定义某个宏
-I<dir> 添加头文件搜索路径
